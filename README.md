# PDP Generator

Система управления ассессментами и генерации индивидуальных планов развития (ИПР) для сотрудников.

## Возможности

- **Управление ассессментами** -- создание, назначение участников, смена статусов (запланирован, в процессе, завершён, отменён)
- **Проведение ассессментов** -- загрузка результатов из Excel или ввод оценок вручную
- **Генерация ИПР** -- автоматическое создание Word-документа с вопросами и заданиями по слабым темам
- **Хранение ИПР** -- сгенерированные файлы сохраняются на сервере и доступны для повторного скачивания
- **Дашборд** -- личная страница с ассессментами и планами развития
- **Управление пользователями** -- создание пользователей, назначение ролей

## Роли

| Роль | Возможности |
|------|-------------|
| **Пользователь** (USER) | Видит свои ассессменты и ИПР, редактирует профиль |
| **Асессор** (ASSESSOR) | Всё что может пользователь + создаёт ассессменты, проводит их, управляет пользователями |

## Требования

- Node.js 18+
- npm

## Установка и запуск

### 1. Установить зависимости

```bash
npm install
```

### 2. Настроить переменные окружения

Создайте файл `.env.local` в корне проекта:

```env
NEXTAUTH_SECRET=ваш-секретный-ключ-для-jwt
NEXTAUTH_URL=http://localhost:3000
```

> Для `NEXTAUTH_SECRET` используйте любую случайную строку. В продакшене замените на надёжный ключ.

### 3. Создать базу данных

```bash
npx prisma db push
```

Это создаст файл `data/pdp.db` (SQLite) со всеми таблицами.

### 4. Создать администратора

```bash
npx tsx prisma/seed.ts
```

Будет создан пользователь с ролью ASSESSOR:
- **Email:** `admin@pdp.local`
- **Пароль:** `admin123`

### 5. Запустить приложение

```bash
npm run dev
```

Приложение будет доступно по адресу [http://localhost:3000](http://localhost:3000).

### 6. Сборка для продакшена

```bash
npm run build
npm start
```

## База данных

Приложение использует SQLite через Prisma ORM. Файл БД: `data/pdp.db`.

### Модели

- **User** -- пользователи (имя, email, роль, грейд, проект, руководитель)
- **Assessment** -- ассессменты (название, статус, грейд, даты)
- **AssessmentParticipant** -- участники ассессмента (оцениваемый / асессор)
- **AssessmentResult** -- результаты (категория, оценка, комментарий)
- **Pdp** -- записи сгенерированных ИПР (файл, статус, привязка к ассессменту)

### Полезные команды

```bash
# Сбросить БД и создать заново
npx prisma db push --force-reset
npx tsx prisma/seed.ts

# Открыть Prisma Studio (визуальный редактор БД)
npx prisma studio

# Сгенерировать Prisma Client после изменения схемы
npx prisma generate
```
